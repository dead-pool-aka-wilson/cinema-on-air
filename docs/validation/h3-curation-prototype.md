# H3 큐레이션 프로토타입 설계 — 태그 기반 발견 효용성 검증

## 1. 개요

| 항목 | 내용 |
|------|------|
| **목적** | 태그 기반 추천이 랜덤 추천 대비 관객 만족도를 높이는지 검증 |
| **방법** | Notion DB에 단편영화 100편 수동 태깅 후 시네필 10명에게 맞춤 추천 제공 |
| **성공 기준** | 태그 기반 추천 만족도가 랜덤 추천 대비 **2배 이상** 높음 |
| **도구** | Notion Database |

## 4. 테스트 방법론

### 4.1 참여자 모집
- **대상**: 단편영화에 관심 있는 시네필 10명
- **모집 채널**: DC Inside 누벨바그 갤러리, 트위터 영화 커뮤니티
- **조건**: 단편영화를 월 1편 이상 감상하는 사람

### 4.2 테스트 절차

**Step 1: 사전 취향 조사 (5분)**
- 선호 장르 3개 선택
- 선호 무드 3개 선택
- 선호 테마 3개 선택
- 최근 인상 깊게 본 단편영화 1편 (오픈형)

**Step 2: 랜덤 추천 (대조군)**
- DB에서 무작위로 5편 추천
- 각 작품 시청 후 만족도 평가 (1~5점)
- 추천 이유 없이 제목과 시놉시스만 제공

**Step 3: 태그 기반 추천 (실험군)**
- Step 1 취향 데이터 기반으로 5편 추천
- 각 작품 시청 후 만족도 평가 (1~5점)
- 추천 이유 ("당신이 좋아하는 #무드 #테마 작품입니다") 함께 제공

**Step 4: 비교 평가 (5분)**
- 두 방식 중 어느 쪽이 더 만족스러웠는지
- 태그 기반 추천의 정확도 평가
- 개선 제안 (오픈형)

### 4.3 측정 지표

| 지표 | 측정 방법 | 성공 기준 |
|------|----------|---------|
| **랜덤 추천 평균 만족도** | 5편 평균 점수 | 기준값 |
| **태그 추천 평균 만족도** | 5편 평균 점수 | 랜덤 대비 **2배 이상** |
| **추천 정확도 체감** | 5점 척도 | 4점 이상 |
| **태그 시스템 유용성** | 5점 척도 | 4점 이상 |

## 5. Notion DB 초기 데이터 수집 계획

**100편 수집 방법:**
1. 부산국제단편영화제 역대 수상작 (공개 목록)
2. 서울독립영화제 역대 상영작 (공개 목록)
3. 유튜브 공개 단편영화 (감독 공식 채널)
4. 비메오 Staff Pick 한국 단편
5. 누벨바그 갤러리 추천 작품 목록

**태깅 기준:**
- 편집팀 1인이 직접 시청 후 태깅
- 장르/무드/테마 각 최소 1개, 최대 3개
- 시청 가능한 작품만 포함 (링크 필수)

## 6. 예상 결과 시나리오

| 시나리오 | 결과 | 해석 |
|---------|------|------|
| **성공** | 태그 추천 만족도 ≥ 랜덤 × 2 | 태그 시스템이 발견 경험을 유의미하게 개선함 → MVP에 태그 시스템 포함 확정 |
| **부분 성공** | 태그 추천 만족도 > 랜덤, but < 2배 | 태그 시스템이 도움이 되지만 개선 필요 → 태그 체계 정교화 후 재검증 |
| **실패** | 태그 추천 만족도 ≤ 랜덤 | 태그 시스템의 효과 없음 → 큐레이션 방식 재검토 (에디터 선별 중심으로 전환) |
